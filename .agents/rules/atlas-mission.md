# Atlas-Docs Mission & Strategy

**Mission:** This repository (`atlas-docs`) serves as the central strategic hub and documentation ledger for the entire Atlas ecosystem. It is the single source of truth for "The What" and "The Why."

## Architectural Boundaries
- **Strategic Definition Only:** This repository contains definitions, goals, personalities, and learnings. 
- **No Execution:** Strictly forbidden are executable scripts, code, automation payloads, or `.agents/workflows`. All implementation ("The How") belongs in operational repositories like `atlas-claw`.
- **Ledger Maintenance:** Every new Project or Agent must have a dedicated sub-folder and an entry in the corresponding root Ledger (e.g., `Agents-Ledger.md`) to serve as a Table of Contents.
